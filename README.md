# CrisisConnect: Unified Emergency Response Platform

CrisisConnect is a state-of-the-art disaster management and emergency response system designed to connect citizens, first responders, and volunteers in real-time. By leveraging modern web technologies, the platform ensures rapid communication and efficient resource allocation during critical situations.

## 🚀 Features

- **Live Incident Map**: Real-time visualization of emergency alerts using interactive maps.
- **Instant SOS**: One-click emergency reporting system for immediate assistance.
- **Mission Control**: Dedicated dashboard for volunteers to track live requests and deploy to critical zones.
- **Resource Management**: Efficient tracking of donations, supplies, and responder availability.
- **AI-Driven Analytics**: Data-driven insights into incident trends and response efficiency.
- **Automated Reporting**: Generation of detailed incident reports for better documentation and analysis.

## 🛠 Tech Stack

- **Frontend**: 
  - [React](https://reactjs.org/) (Vite)
  - [Tailwind CSS](https://tailwindcss.com/) for modern, responsive UI.
  - [Lucide React](https://lucide.dev/) for intuitive iconography.
  - [React Leaflet](https://react-leaflet.js.org/) for real-time mapping.
  - [Recharts](https://recharts.org/) for analytical data visualization.
- **Backend**:
  - [Node.js](https://nodejs.org/) & [Express](https://expressjs.com/)
  - [SQLite](https://www.sqlite.org/) for lightweight, reliable data storage.
- **Infrastructure**:
  - [Docker](https://www.docker.com/) & [Docker Compose](https://docs.docker.com/compose/) support for containerized deployment.

## 🚦 Getting Started

### Prerequisites

- Node.js (v18 or higher)
- npm (v9 or higher)

### Setup & Installation

1. **Clone the repository**:
   ```bash
   git clone <repository-url>
   cd "Bhuvanesh Digital Dynamos"
   ```

2. **Install Root Dependencies**:
   ```bash
   npm install
   ```

3. **Install Backend Dependencies**:
   ```bash
   cd backend
   npm install
   cd ..
   ```

4. **Run the Application**:
   You can start both the backend and frontend simultaneously:
   - To start the backend: `npm run backend`
   - To start the frontend: `npm run dev`
